INPUT CATEGORIES AND OPTIONS:

CATEGORY 1: User_type
This category represents the type of user asking the question.

Option 1.1 - "standard": Standard consumer customer - normal, polite tone
Option 1.2 - "bulk_enterprise": Bulk/enterprise buyer - professional, may mention company/volume needs
Option 1.3 - "frustrated": Frustrated customer

CATEGORY 2: Information_type
This category represents the type of information the user is seeking.

Option 2.1 - "product": Product question (availability, pricing, recommendations)
Option 2.2 - "policy": Policy question (returns, shipping, ordering, company info)
Option 2.3 - "out_of_scope": Out-of-scope request (order placement, tracking, returns processing, account changes, tech support)

CATEGORY 3: Question_type
This category represents the clarity and structure of the question.

Option 3.1 - "clear": Clear and specific request - well-articulated question about one topic
Option 3.2 - "vague": Vague or ambiguous - unclear what they're asking for, but still one question
Option 3.3 - "multi_intent": Multiple questions or intents in one query - this is the ONLY case where multiple questions are allowed
